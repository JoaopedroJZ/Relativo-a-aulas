Formularios

1. O que é um Formulario?

Um formulario nada mais é que um lugar onde o usario faz um cadastro ou um login, os formularios podem ser preenchidos tanto de forma fisica quanto de forma digital.

2. Como fazer um Formulario?

Todo o formulario deve estar dentro de uma tag "FORM", para fazer as "caixinhas" onde você preenche com nome, senha, data de nascimento e etc, basta você usar uma tag chama "INPUT", o input tem MUITAS definições dentre elas para o NOME você utiliza o com a forma de "INPUT type TEXT" dentro dessa tag terá duas especificações, NAME e ID, lá você coloca oque você quer cobrar do usario seja NOME ou SOBRENOME. Como desligar o auto preenchimento do input? vá na tag "form" e coloque "autocomplete: off".

3. Label, Name e Id

O seu navegador é burro, ele não sabe diferenciar um SOBRENOME de um NOME pra isso utilizamos LABEL que é um identificador, em linguagens de programação isso é importante pois quando você ENVIA o dado ele vai ser identificado por Label, Name e Id. Sendo em PHP mais comum o Name e em javascript o ID. O label é basicamente uma etiqueta que vai identificar oque é oque dentro do seu formulario, sempre antes do input você vai colocar algo pra pessoa identificar oque ela vai colocar, utilizamos o label para o COMPUTADOR identificar que X texto é pra Y input e vice-verse, dentro da tag label tem uma especificação que é o "for" no for você coloca o ID ou o NAME, a importancia disso é identificar e facilitar para os navegadores.

4. Pra onde é enviado as informações?

Normalmente se manda os dados para uma linguagem de programação sendo as duas mais comuns Java e PHP (backend), pra isso coloque um "action" no seu form no action você coloca o arquivo da linguagem que você estiver usando.

5. Envios

Existem duas formas de envio de formulario, GET e POST, sendo o padrão o GET, para mudar o metodo basta colocar a tag METHOD dentro do form. o metodo GET faz com que as coisas registradas na caixa de envio sejam mostradas no URL do seu site e o POST faz eles não aparecerem oque torna seu site mais seguro, porém eles ainda sim estão visiveis só não no URL, para isso você vai no inspecionar do seu site e depois vá em network e clique no arquivo que você registrou como enviar e depois em header e pronto la esta seus dados, para criptografar seus dados você teria que utilizar outras coisas. situações os quais você não pode usar o "GET", em formularios de mais de 3 mil linhas, formularios de senha, envios de foto/arquivos e etc. Porque usar o "POST", conteudos sensiveis como senhas para proteger mais você teria que estudar HTTPS.

6. Outros Inputs

Existem outros Inputs que você deve utilizar para criar um formulario sendo eles, senha, reset e entre outros que você ainda vai aprender. assim como existem outros inputs também existem outros valores que são eles o "required" que faz com que você seja obrigado a preencher determinado input ou campo, e os maxlenght e minlenght que são aqueles que determinam o maximo e o minimo de letras.Size serve para colocar o tamanho da caixa de input sendo o numero que você colocar o numero de letras. o placeholder é algo que segura o lugar até o usario clicar. autocomplete no on ou no off serve para ligar o preenchimento automatico isso no form pois se você colocar o autocomplete no input você vai dizer para que aquele determinado input serve e o que o auto preenchimento do google deve preencher. Voltando para os Inputs, o input NUMBER dentro deste input você só pode colocar numeros, com isso também descobrimos valores que são eles o max e o min, o max determina o numero maximo que você pode colocar ali e o min é auto explicativo, entretanto se caso você queira colocar 0.5 você tem que colocar um valor chamado STEP que basicamente é em quanto em quanto que os numeros irão contar, é recomendado que você coloque 0.1. Temos o input Month que nada mais que preenche o campo pela data, é imporante dizer que o value dentro desse input caso você queira colocar um mês deve ser colocado ANO-MÊS (2024-06), do mesmo jeito que existe o de mês existe o de dia que no caso é o DATE sendo o valor ANO-MÊS-DIA, também temos em horas que seria o input TIME.

7. Compatibilidades

Existem diversos navegadores e alguns podem fazer com que seu formulario seja mostrado de formas diferentes, isso acontece por conta de compatibilidade, pois alguns navegadores tem diferentes jeitos de mostrar diferentes, por exemplo, o firefox pode mostrar um formulario diferente do seu, isso também vale pros celulares.
